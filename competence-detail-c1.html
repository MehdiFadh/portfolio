Compétence C1 : RÉALISER

Développer des applications informatiques (Niveau 1)

Contexte : Application de Gestion des Pointages (Thimon)

Dans le cadre de mon alternance chez Thimon, j'ai eu pour mission principale de développer une application complète de gestion des pointages et des demandes d'absence pour les salariés. Ce projet m'a permis de mettre en œuvre la compétence RÉALISER dans un contexte professionnel réel.

Partie 1 : Analyse des besoins et Prise en main

Le projet a débuté par une phase d'analyse. Il fallait comprendre les besoins du service RH : suivre les heures d'arrivée et de départ, gérer les congés payés et les RTT, et automatiser les calculs mensuels. J'ai dû me familiariser avec l'environnement technique imposé : Windev et le SGBD HFSQL. Cette étape a nécessité d'étudier la documentation et de réaliser des prototypes simples (AC11.01) pour valider la faisabilité des fonctionnalités demandées.

Partie 2 : Conception et Développement de l'Interface

Une fois les besoins clarifiés, j'ai entamé la phase de conception. L'objectif était de créer une interface intuitive pour que les employés puissent pointer rapidement. J'ai conçu des maquettes d'écrans (AC11.02) avant de les implémenter. L'interface comprend un tableau de bord pour l'utilisateur (solde de congés, historique) et un module d'administration pour les RH. Le développement m'a amené à créer des composants graphiques interactifs (boutons, calendriers) et à gérer les événements utilisateurs (clics, validation de formulaires) pour offrir une expérience fluide (AC11.04).

Partie 3 : Logique Métier et Base de Données

Derrière l'interface, j'ai développé toute la logique de traitement. Cela impliquait la connexion à la base de données HFSQL pour stocker les pointages en temps réel. J'ai écrit des algorithmes pour calculer automatiquement la durée de travail journalière et alerter en cas d'anomalie. Cette rigueur dans le code a été essentielle pour assurer la fiabilité des données de paie.

Partie 4 : Tests et Déploiement

Avant la mise en production, j'ai réalisé une phase de tests unitaires et fonctionnels (AC11.03). J'ai simulé différents scénarios (oubli de pointage, congés chevauchant des week-ends) pour corriger les bugs. L'application a ensuite été déployée sur le serveur Windows de l'entreprise et est aujourd'hui utilisée quotidiennement par l'équipe.
