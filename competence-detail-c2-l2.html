Compétence C2 : OPTIMISER

Optimiser les applications informatiques (Niveau 2)

Contexte : Gestion de Parc Informatique (SAE 3.01)

Lors de la SAÉ 3.01, j'ai participé à la conception et au développement d'une application web de gestion de parc informatique en PHP/MySQL. Au-delà du développement fonctionnel, ce projet nécessitait une réflexion approfondie sur la structure des données pour garantir la cohérence et la performance lors de l'interrogation d'un grand nombre d'équipements.

Partie 1 : Choix des Structures de Données (AC22.01)

Le premier défi était de modéliser les relations complexes entre les tickets d'incidents, les utilisateurs et les différents types de matériel (PC, Imprimantes, Serveurs). Pour éviter la redondance et les anomalies de mise à jour, j'ai appliqué les règles de normalisation (3NF) à notre base de données relationnelle. J'ai choisi d'utiliser des tables de jointure pour les relations "Many-to-Many" (ex: un logiciel peut être installé sur plusieurs postes, et un poste a plusieurs logiciels), ce qui nous a permis de structurer les données de manière optimale pour les futures évolutions.

Partie 2 : Techniques Algorithmiques et Optimisation (AC22.02)

Pour la recherche de matériel, nous avons été confrontés à des temps de réponse qui augmentaient avec le volume de données. J'ai mis en place des techniques avancées comme l'ajout d'index sur les colonnes fréquemment recherchées (numéro de série, nom d'utilisateur). Côté code, j'ai implémenté un système de pagination côté serveur (LIMIT/OFFSET en SQL) plutôt que de récupérer tous les enregistrements pour les filtrer en PHP, réduisant drastiquement l'empreinte mémoire du script.

Partie 3 : Optimisation de Code Existant (AC22.03)

Nous avons repris une partie du code existant qui souffrait du problème "N+1 queries" (une requête pour la liste, puis une requête par élément pour récupérer les détails). J'ai refactorisé ces sections en utilisant des jointures SQL (INNER JOIN / LEFT JOIN) pour récupérer toutes les informations nécessaires en une seule requête optimisée. Cette analyse et cette réécriture ont permis de diviser par 10 le nombre de requêtes envoyées à la base de données sur la page d'accueil du tableau de bord.
