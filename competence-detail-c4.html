Compétence C4 : GÉRER

Gérer des données de l'information (Niveau 1)

Contexte : Base de Données Vinotrip (SAÉ)

Le projet Vinotrip consistait à développer une plateforme de réservation de séjours oenotouristiques. Un élément central de cette application était la gestion des données : utilisateurs, commandes, paniers et catalogue de séjours. Pour répondre à ce besoin, j'ai mis en place une base de données relationnelle robuste utilisant le SGBD PostgreSQL. Ce travail valide le Niveau 1 de la compétence GÉRER.

Partie 1 : Conception et Mise en place (AC14.01)

La première étape a été de modéliser les données. J'ai réalisé le Modèle Conceptuel de Données (MCD) pour identifier les entités (Client, Séjour, Etape) et leurs relations. J'ai ensuite traduit ce modèle en Modèle Logique de Données (MLD), puis en script SQL de création. J'ai utilisé des commandes CREATE TABLE pour structurer la base, en définissant rigoureusement les clés primaires, les clés étrangères pour les liens entre tables, et les contraintes d'intégrité (AC14.01).

Partie 2 : Exploitation des Données (AC14.02)

Pour rendre l'application fonctionnelle, j'ai dû écrire de nombreuses requêtes SQL. J'ai développé des requêtes pour :

  • L'authentification des utilisateurs (SELECT avec clause WHERE).
  • La recherche de séjours selon des critères (prix, région).
  • La gestion du panier et la validation des commandes (INSERT, UPDATE).
L'utilisation de jointures (INNER JOIN, LEFT JOIN) a été essentielle pour rassembler les informations dispersées dans plusieurs tables (ex: détails d'une commande avec infos client et séjour) (AC14.02).

Analyse et Retour d'Expérience

Difficultés rencontrées

La quantité de notions à assimiler en peu de temps (Normalisation, syntaxe SQL stricte) a rendu l'évaluation complexe.

Points de facilité

Bonne compréhension intuitive des concepts de bases de données relationnelles et logique du langage SQL.
